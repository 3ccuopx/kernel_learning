Основные сведения

 

Монолитное ядро (Линус Товальдс) -  модули и ядро в одном адресном пространстве => выше скорость, меньше надежность.

Микроядра (Эндрю Танненбаум) - вся начинка общается в режиме клиент-сервер => меньше скорость, больше надежность.

 

           Отличия пространства пользователя от пространства ядра:

1.   У ядра свое собственное API, оно не может взаимодействовать с билиотеками какого-либо языка и наоборот. Функции ядерного API могут писаться также как и библиотечные, например, sprint, но реализация у них совершенно другая.

2.   Отсутствие защиты памяти, так как ядро имеет доступ к любой памяти в любой момент. Ошибка в модуле ядра может моментально крашнуть всю систему.

3.   В ядре нельзя использовать вычисления с плавающей точкой.

4.   Стек (размер занямаемой памяти) локальной переменной ядра строго фиксирован и занимает 16КБ для 64-битной архитектуры процессора.

5.   Ядро активно обновляется и содержит много функций без документации.

 

           Особенности файловой системы Linux:

/dev - взаимодействие с устройствами.

/lib/modules/`uname -r`/build/include — содержит все хедер-файлы ядра для включения определений в код модуля

/proc - специальные файлы модулей, содержащие отладочную или диагностическую информацию.

    /proc/meminfo    — сводная информация о использовании памяти в системе;

    /proc/devices    — список дpайвеpов устpойств, встpоенных в действующее ядpо;

    /proc/dma     — задействованные в текущий момент каналы DMA;

    /proc/filesystems  — файловые системы, встpоенные в ядpо;

    /proc/interrupts   — список задействованных в текущий момент пpеpываний;

    /proc/ioports      — список задействованных в текущий момент поpтов ввода/вывода;

    /proc/version    — версия ядра (из этого источника черпает информацию команда uname):

/sys - файловая система

 

           Процесс запуска ОС:

1. Запуск BIOS/UEFI.

    а) Происходит инициализация (обнаружение, проверка и настройка) аппаратных компонентов компьютера (процессор, память, диски, сеть) посредством сканирования шин PCI и USB.

    б)  BIOS загружает первый сектор первого загрузочного диска в ОЗУ и передает ему управление.

2.  Первый этап загрузчика. Загруженный первый сектор загружает в память код второго (более крупного) этапа загрузчика и передает ему управление.

3.  Второй этап загрузчика.

    а) Сначала получает данные об имеющихся на диске ОС. Принимает в качестве параметров файлы сжатого образа ядра vmlinuz-... и файл сжатого образа initrams. Частично их     распаковывает.

    б)  Переходит к точке входа в ядро. Отображается меню запуска GRUB позволяющее выбрать версию ОС и ядра.

    в)  После выбора ОС, ядро загружается в ОЗУ и ему передается управление.

4.  Запуск ядра.

    а)  Ядро инициализирует аппаратную и программную среду (независимо от BIOS и GRUB).

    б)  Ядро находит и распаковывает (если еще не распакован) образ initramfs в оперативной памяти.

    в)  После распаковки ядро монтирует образ initramfs как временную корневую файловую систему в оперативной памяти.

5.  Запуск initramfs.

    а)  Запускаются скрипты, выполняющие загрузку необходимых модулей ядра в ОЗУ (в т.ч. f2fs.ko).

    б)  Монтирование реальной корневой файловой системы.

    в)  Размонтирование временной файловой системы и освобождение занимаемой ею памяти.

    г)  Передача управления ядру с оформленной корневой системой в постоянной памяти.

6.  Вызов ядром первого процесса /sbin/init, который инициализирует систему согласно конфигурации.

 

           Основные команды работы с модулями:

insmod     - загрузка модуля без проверок взаимосвязей. Принимает имя файла модуля.

modprobe   - загрузка модуля с проверкой взаимосвязей с другими модулями (они будут подгружены первыми). Принимает ID модуля или его емя файла.

depmod -a - обновляет файлы каталога /lib/modules/'uname -r'

rmmod      - выгрузка ранее загруженного модуля модуля (при условии отсутствия взаимосвязи с другими модулями)

 

           Работа с Linux:

sudo systemctl isolate multi-user.target    - перевод в терминальный режим

sudo systemctl isolate graphical.target     - перевод в графический режим

sync; sudo reboot                           - завершение процессов и перезагрузка ОС

sudo shutdown -h now                         - завершение работы

 

Прерывания

Прерывания делят на синхронные и асинхронные.

·               Синхронные прерывания (исключения) – порождаются процессором или выполняемым кодом. Возникают в определенный момент времени выполнения кода. Пример: ошибка деления на 0, системный вызов. Поведение: процессор приостанавливает выполнение программы, обрабатывает исключение и пытается вернуться к той же инструкции.

·               Асинхронные прерывания (аппаратные прерывания) – порождаются внешними устройствами (переферией). Возникают в случайный, не зависящий от программы момент времени. Пример: нажатие на клавишу. Поведение: процессор заканчивает текущую инструкцию, обрабатывает прерывание, а затем переключается на следующую инструкцию.

 

       

Модули ядра

 

 

Данные о модулях в системе

/boot/System.map-`uname -r` - Статическая таблица всех имен ядра (при его сборке)

/proc/kallsyms - Динамическая таблица имен ядра (с учетом подгруженных)

y - включен в состав ядра

m -  включен в качестве подгружаемого модуля

n - не включен в сборку

 

Шапка модуля

MODULE_LICENSE("GPL")                    лицензия модуля (в открытом доступе или нет)

MODULE_AUTHOR("IVAN")                    автор модуля

MODULE_DESCRIPTION("A sample driver")    описание модуля

 

Параметры модулей ядра

                       Передача параметров из терминала:

sudo insmod ./modparams1.ko mp_debug_level=2

или

sudo sh -c "echo 0 > /sys/module/modparams1/parameters/mp_debug_level"

 

            Объявление параметров в файле модуля:

static int mp_debug_level;                              Объявление переменной

module_param(mp_debug_level, int, 0660);                Возможность считывания параметра с терминала

MODULE_PARM_DESC(mp_debug_level,"Уровень отладки");     Информация, которая будет доступна в modinfo -p

   

module_param(

        mp_debug_level     имя параметра

        int                тип данных переменной (обязательно с модификатором static)

        0660               права доступа (видимость через sysfs)

);

MODULE_PARM_DESC(         Информация о параметре, которая будет доступна в modinfo -p

        mp_debug_level,    имя параметра

        "Уровень отладки" текстовое описание

);

 

Стандартные типы данных

byte, hexint, short, ushort, int, uint, long, ulong

charp: указатель на символ (строка)

bool: логический тип, значения 0/1, y/n, Y/N.

invbool: то же самое, но с обратной логикой (N = true).

 

Автозагрузка модулей

 

1. Чтобы реализовать автозагрузку модулей, необходимо добавить цель install в Makefile. Это позволяет при вводе команды sudo make install скопировать скомпилированный модуль в системный каталог /lib/modules/$(uname -r)/…/.

 

install:

    sudo make -C $(KDIR) M=$(PWD) modules_install

2. Затем нужно добавить/изменить конфигурационный файл загрузки модуля: /etc/modules-load.d/min_sysinfo.con, добавив в него строчку-название модуля min_sysinfo.

3. Перезапустить систему.
